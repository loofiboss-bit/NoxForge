# NoxForge 2.0.0

NoxForge 2.0.0 rebuilds the complete Fedora KDE 44 visual system while keeping
the established graphite/lime Industrial Precision identity and existing
package identifiers.

## Highlights

- Schema v3 design tokens generate the Qt palette, KDE color schemes, Plasma
  colors, physical QML token copies, and the canonical angled N/F mark.
- The Plasma 6.7 contract covers all 43 widget families plus weather artwork,
  oriented edge states, margins, tile hints, and blur masks.
- The Qt 6 style shares one focus/selection model across controls and implements
  matching LTR/RTL hit geometry for scrollbars, sliders, combo boxes, and spin
  boxes.
- State icons and cursors use semantically distinct original glyphs. Wait and
  progress cursors contain 12 frames at 80 ms for 24, 32, and 48 px.
- SDDM, Splash, Logout, and Alt+Tab use generated tokens and the shared mark.
  SDDM includes visible field labels, keyboard focus, accessible names, a stable
  status row, session selection, and distinct power-action feedback.
- Wallpaper outputs cover 2560x1440, 3840x2160, and 3440x1440; SDDM uses a
  deterministically dimmed variant.

## Verification status

The stable release passed automated validation, 43 Python tests, 11 CTest cases,
QML linting, deterministic generation, render-matrix captures, repeated isolated
user/system install cycles and a twice-built byte-reproducible source archive.
The GitHub release publishes that archive with its SHA-256 checksum.

Interactive Plasma/Wayland and real SDDM checks remain explicitly Pending until
performed after deliberately applying the theme in a live session; structural
or offscreen rendering is not counted as a substitute.

## Installation

The source archive contains reversible user and system installers. Neither
installer applies the theme, changes the panel, edits SDDM configuration or
restarts Plasma.

```bash
python3 scripts/build.py
./scripts/install.sh --user
sudo ./scripts/install-system.sh --system
```
